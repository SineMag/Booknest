Reviews API Documentation

Route: /reviews

üìò Introduction

The /reviews API allows users to create, view, update, and delete reviews for accommodations on the BookNest platform. This document explains each endpoint, what it expects, and what it returns.

üß© Data Model

A review has the following structure:

{
  id: number,
  starRatings: number,
  reviewer: string,
  reviewText: string,
  createdAt: Date,
  userId: number,
  accomodationId: number
}

GET /reviews/:id

Returns a single review using its ID.

‚úî Example

GET /reviews/1

GET /reviews/user/:id

Returns all reviews created by a specific user.

‚úî Example

GET /reviews/user/3

GET /reviews/accommodation/:id

Returns all reviews for a specific accommodation.

‚úî Example

GET /reviews/accommodation/12

POST /reviews

Creates a new review and returns the newly created record.

‚úî Example Request Body

{
  "starRatings": 4,
  "reviewer": "Alice Smith",
  "reviewText": "Nice place!",
  "userId": 2,
  "accomodationId": 5
}

‚úî Example Response

{
  "id": 12,
  "starRatings": 4,
  "reviewer": "Alice Smith",
  "reviewText": "Nice place!",
  "createdAt": "2025-01-18T12:00:00.000Z",
  "userId": 2,
  "accomodationId": 5
}

PUT /reviews

Updates an existing review. The id must be included in the request body.

‚úî Example Request Body

{
  "id": 12,
  "starRatings": 5,
  "reviewer": "Alice Smith Updated",
  "reviewText": "Amazing stay!"
}

‚úî Example Response

{
  "id": 12,
  "starRatings": 5,
  "reviewer": "Alice Smith Updated",
  "reviewText": "Amazing stay!",
  "createdAt": "2025-01-18T12:00:00.000Z",
  "userId": 2,
  "accomodationId": 5
}

DELETE /reviews

Deletes a review using the id provided in the body.
